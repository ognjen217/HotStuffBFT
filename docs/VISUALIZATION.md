# Log visualization

The simulator saves the same text that is printed in the terminal to a scenario-specific log file:

```bash
go run ./cmd/hotstuff-sim --scenario happy --timeout-ms 1000
```

Default outputs:

```text
logs/happy.txt
visualizations/happy.html
```

The HTML file is generated by the Python script:

```bash
python3 scripts/visualize_log.py --log logs/happy.txt --out visualizations/happy.html
```

The visualization contains:

- summary cards with event counts, number of QCs, decisions, timeouts, and safety rejects;
- a replica-lane timeline where each replica has its own lane;
- colored markers for proposals, QCs, locks, decisions, invalid executions, timeouts, and Byzantine actions;
- an important-events table extracted from the raw log;
- the final state summary copied from the simulator output.

The script uses only the Python standard library, so it does not require matplotlib, pandas, graphviz, or any other external package.

## Useful commands

Run and visualize all built-in scenarios:

```bash
for scenario in happy silent-leader byzantine-equivocation banking-block-transfer delayed-network; do
  go run ./cmd/hotstuff-sim --scenario "$scenario" --timeout-ms 1000
done
```

Disable visualization but still save logs:

```bash
go run ./cmd/hotstuff-sim --scenario happy --visualize=false
```

Disable log files completely:

```bash
go run ./cmd/hotstuff-sim --scenario happy --log-dir ""
```

Change output directories:

```bash
go run ./cmd/hotstuff-sim --scenario happy --log-dir demo-logs --viz-dir demo-viz
```
